# Plan: Syntax Highlight for .gmi Files

## Goal

Add syntax highlighting for code blocks in `.gmi` (Gemtext) files using language identifiers specified in preformatted block alt text.

## Background

- `.gmi` files are processed by `GeminiEngine` in `gemini.ts`, which currently calls `buffer()` from Dioscuri to convert gemtext to HTML
- Syntax highlighting via `rehypeStarryNight` is applied to `.html` files through the remark pipeline in `_config.ts`
- Dioscuri's `buffer()` already outputs preformatted blocks with language identifiers as `<pre><code class="language-bnf">...</code></pre>`
- The build output confirms `buffer()` generates correct HTML structure with `class="language-bnf"` for ` ```bnf ` blocks
- `rehype-starry-night` looks for `code` elements with `language-*` classes, which `buffer()` already provides

## Implementation

### Overview

Modify `gemini.ts` to process the HTML output from `buffer()` through `rehype-starry-night`, which will apply syntax highlighting to code blocks with language identifiers.

### Detailed Steps

1. **Modify `gemini.ts`**:
   - Keep using `buffer()` from Dioscuri to generate initial HTML (it already outputs correct `<pre><code class="language-*">` structure)
   - Import `rehype-starry-night` and `all` grammars from `@wooorm/starry-night`
   - Import `rehype` to create a processor that can apply rehype plugins
   - Change `GeminiEngine.renderComponent()` to:
     1. Call `buffer(content)` to get initial HTML string
     2. Process the HTML string with a `rehype` processor:
        - Parse HTML to HAST (may need `fragment: true` option for HTML fragments)
        - Apply `rehype-starry-night` plugin with `grammars: all`
        - Serialize back to HTML string
     3. Return the processed HTML

2. **Handle edge cases**:
   - Pre blocks without language identifiers: `rehype-starry-night` will skip them (no `language-*` class)
   - Unknown language identifiers: plugin logs warning but doesn't fail
   - Empty preformatted blocks: skip highlighting

### Technical Details

```typescript
import { buffer } from "https://esm.sh/dioscuri@1.3.0";
import rehypeStarryNight from "https://cdn.jsdelivr.net/npm/rehype-starry-night@2.2.0/+esm";
import { all } from "https://cdn.jsdelivr.net/npm/@wooorm/starry-night@3.6.0/+esm";
import { rehype } from "https://esm.sh/rehype@13?target=es2022";

export default class GeminiEngine implements Engine {
  async renderComponent(content: string, _data?: Record<string, unknown>, _filename?: string) {
    // Step 1: Generate HTML with dioscuri (already includes class="language-*")
    const html = buffer(content);

    // Step 2: Process HTML through rehype-starry-night for syntax highlighting
    const result = await rehype()
      .use(rehypeStarryNight, { grammars: all })
      .process(html);

    return result.toString();
  }
}
```

**Note**: May need to pass `{ fragment: true }` to the HTML parser if `rehype` doesn't correctly parse HTML fragments (output from `buffer()` is not a full HTML document).
